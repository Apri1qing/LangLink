📄 产品文档（PRD v1.0）

🧭 一、产品概述

1.1 产品名称（暂定）

TravelTalk（可后续再改）

⸻

1.2 产品定位

一个极简的旅行翻译工具，通过“语音 + 图片 + 常用语”，帮助用户在陌生语言环境中完成即时沟通。

⸻

1.3 核心价值
	•	✅ 不需要切页面（单页完成所有操作）
	•	✅ 降低沟通成本（按住说即可）
	•	✅ 支持“看图沟通”（菜单 / 路牌 / 标识）
	•	✅ 即使不会外语，也能自然交流

⸻

1.4 目标用户
	•	出境游客（重点：中国 → 日本）
	•	不会外语的人群
	•	临时跨语言沟通需求用户

⸻

🎯 二、核心使用场景

场景1：问路
	1.	用户说：“地铁站在哪里”
	2.	自动翻译为日语并播放
	3.	对方回答
	4.	用户录音 → 翻译为中文

⸻

场景2：看菜单
	1.	拍照菜单
	2.	自动翻译
	3.	用户说：“这个是什么”
	4.	对方回答 → 翻译

⸻

场景3：购物
	•	点击“多少钱”
	•	自动播放目标语言
	•	对方回答 → 翻译

⸻

🎤 四、语音翻译模块

4.1 功能描述

	•	母语 → 目标语言
	•	目标语言 → 母语

⸻

4.2 交互设计

UI结构

[ 中文 🎤 ]   [ 日文 🎤 ]


⸻

用户操作流程

中文 → 日文
	1.	点击左侧按钮
	2.	开始录音
	3.	再次点击结束
	4.	自动：
	•	识别文本
	•	翻译
	•	播放语音

⸻

日文 → 中文

同理（右侧按钮）

⸻

4.3 输出内容
	•	原文（小字）
	•	翻译（大字）
	•	自动语音播放

	•	不占用空间

翻译完成后：
	•	弹出覆盖层（Overlay）
	•	显示大字体翻译
	•	自动播放语音
	•	2~4秒后自动消失

⸻

4.5 异常处理
	•	识别失败 → 提示“未识别到语音”
	•	网络失败 → 提示“网络异常”
